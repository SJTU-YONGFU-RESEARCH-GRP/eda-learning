## Outcomes

After this module you can: **Encode a slicing floorplan as an H/V slicing tree or polish expression and pack rectangles from it.**

## Two tracks (pick one or both)

### Track A — Implement (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and implement **slicing tree / polish expression packing (H and V cuts)** on `examples/tiny_modules.json`.
2. Complete [CHECKLIST.md](CHECKLIST.md) with legality, area, density, and deadspace (plus lab-specific metrics).
3. Optional self-check: `./scripts/module.sh 02-01 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/slicing-floorplan/](http://127.0.0.1:8080/tools/slicing-floorplan/)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/slicing-floorplan/](https://universal-verification-methodology.github.io/learning/tools/slicing-floorplan/)
3. Tools shelf: open `slicing-floorplan` from the platform tools index
4. Load the **starter modules**, run the packing / search, inspect outline metrics.
5. Check off Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs teach floorplan literacy on tiny outlines — not production place-and-route.
