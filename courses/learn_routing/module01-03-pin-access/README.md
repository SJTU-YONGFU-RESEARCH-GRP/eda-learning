## Outcomes

After this module you can: **Map each pin placement to grid access points usable by detailed routers.**

## Two tracks (pick one or both)

### Track A — Implement (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and implement **pin access point assignment on the routing grid** on `examples/tiny_dr.json`.
2. Complete [CHECKLIST.md](CHECKLIST.md) with track usage and overflow when relevant.
3. Optional self-check: `./scripts/module.sh 01-03 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/pin-access/](http://127.0.0.1:8080/tools/pin-access/)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/pin-access/](https://universal-verification-methodology.github.io/learning/tools/pin-access/)
3. Tools shelf: open `pin-access` from the platform tools index
4. Load the **starter detailed routing** instance, run the router, inspect track overflow and vias.
5. Check off Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs teach detailed routing literacy on a tiny M1/M2 grid — not production signoff routers.
