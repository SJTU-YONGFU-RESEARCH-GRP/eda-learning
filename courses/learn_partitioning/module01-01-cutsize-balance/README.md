## Outcomes

After this module you can: **Compute edge cutsize and part balance for a bipartition and explain the cut–balance trade-off.**

## Two tracks (pick one or both)

### Track A — Implement (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and implement **cutsize + balance metrics on a labeled bipartition** on the tiny graphs.
2. Complete [CHECKLIST.md](CHECKLIST.md) with metrics (cutsize, balance, objective).
3. Optional self-check: `./scripts/module.sh 01-01 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/cutsize-balance/index.html](http://127.0.0.1:8080/tools/cutsize-balance/index.html)
2. Tools shelf: open `cutsize-balance` from the platform tools index
3. Load the **starter graph**, run the algorithm, inspect metrics.
4. Check off Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs teach algorithm literacy on tiny instances — not production PDK flows.
