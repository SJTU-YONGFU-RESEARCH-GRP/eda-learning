## Outcomes

After this module you can: **Build a legal initial bipartition (random, greedy, or grow) that respects balance before refinement.**

## Two tracks (pick one or both)

### Track A — Implement (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and implement **random / greedy / grow initial bipartition** on the tiny graphs.
2. Complete [CHECKLIST.md](CHECKLIST.md) with metrics (cutsize, balance, objective).
3. Optional self-check: `./scripts/module.sh 01-03 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/initial-bipartition/index.html](http://127.0.0.1:8080/tools/initial-bipartition/index.html)
2. Tools shelf: open `initial-bipartition` from the platform tools index
3. Load the **starter graph**, run the algorithm, inspect metrics.
4. Check off Track B items in [CHECKLIST.md](CHECKLIST.md).
