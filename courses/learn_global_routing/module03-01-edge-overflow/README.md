After this module you can: **Report total, max, and count of edge overflows from accumulated route usage.**

## Two tracks (pick one or both)

### Track A — Implement (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and implement **edge overflow metrics (total, max, count)** on `examples/tiny_gr.json`.
2. Complete [CHECKLIST.md](CHECKLIST.md) with edge usage and overflow when relevant.
3. Optional self-check: `./scripts/module.sh 03-01 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/edge-overflow/](http://127.0.0.1:8080/tools/edge-overflow/)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/edge-overflow/](https://universal-verification-methodology.github.io/learning/tools/edge-overflow/)
3. Tools shelf: open `edge-overflow` from the platform tools index
4. Load the **starter global routing** instance, run the router, inspect edge overflow.
5. Check off Track B items in [CHECKLIST.md](CHECKLIST.md).
