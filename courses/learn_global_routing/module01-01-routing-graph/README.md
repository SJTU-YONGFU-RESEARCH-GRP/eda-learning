## Outcomes

After this module you can: **Build the undirected GCell adjacency graph and list edges on a 4×2 grid.**

## Two tracks (pick one or both)

### Track A — Implement (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and implement **GCell routing graph (adjacent tile edges)** on `examples/tiny_gr.json`.
2. Complete [CHECKLIST.md](CHECKLIST.md) with edge usage and overflow when relevant.
3. Optional self-check: `./scripts/module.sh 01-01 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/routing-graph/](http://127.0.0.1:8080/tools/routing-graph/)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/routing-graph/](https://universal-verification-methodology.github.io/learning/tools/routing-graph/)
3. Tools shelf: open `routing-graph` from the platform tools index
4. Load the **starter global routing** instance, run the router, inspect edge overflow.
5. Check off Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs teach global routing literacy on a tiny GCell graph — not production detailed routers.
